# Changelog

## v3.0 — 2026-07-09

CPOS — Collectibles Portfolio Operating System.

- Renamed the project to CPOS and reframed it as a Portfolio Manager instead of a pricing prompt.
- Added the design package (`design/`): Principles, Vision, Architecture, Terminology, Design Decisions, Task Requirements, Roadmap.
- Added `TASK-EDITION.md` — the operational ChatGPT prompt structured into six modules (Initialization, Master Inventory, Market Intelligence, Portfolio Management, Daily Report, Continuous Improvement).
- Added `SPECIFICATION.md` — the complete implementation standard.
- Added `templates/` — Master Inventory and Daily Report templates.
- Added `examples/` — example inventory and example daily report.

## v2.2 — 2026-07-09

- Added the Initialization / First-Time Use flow: accept collections in any format, produce a draft inventory, require explicit confirmation before locking the SSOT.

## v1.0 — 2026-07-09

- First version of the Master Inventory prompt: SSOT rules, market valuation priority, per-card fields, Opportunity Score, S/A/B/C classification, and the 收藏投資組合日報 report format.
